# 已知问题与改进方向

## 当前已知问题

### 1. 天气缓存策略
- **问题**：天气数据缓存后不会自动更新，同一天内多次请求返回相同数据。Open-Meteo API 每天更新一次，但缓存没有过期机制。
- **影响**：如果 API 在某天更新后缓存未被刷新，可能显示过时数据。
- **建议**：添加 `fetched_at` 时间戳检查，如果超过 24 小时则重新获取。

### 2. 任务颜色生成
- **问题**：颜色通过项目名称哈希确定，相同项目名称始终获得相同颜色。如果删除任务后重新创建同名任务，颜色会重复。
- **影响**：不同项目可能哈希到相同颜色（20 色调色板，冲突概率低但存在）。
- **建议**：改为完全随机颜色，或在创建时检测冲突。

### 3. 日程时间冲突检测
- **问题**：冲突检测仅在同一天内进行，跨天日程（如果允许）不会被检测。
- **影响**：正常使用场景下影响不大，因为日程通常为单日事件。
- **建议**：如果未来支持跨天日程，需改进冲突检测逻辑。

### 4. 移动端 PWA 图标
- **问题**：PWA 使用 SVG 图标，部分 Android 设备对 SVG manifest 图标支持不完整。
- **影响**：某些设备上添加到主屏幕后图标可能显示为默认图标。
- **建议**：添加 PNG 格式的 fallback 图标（192x192, 512x512）。

### 5. SQLite 并发写入
- **问题**：better-sqlite3 是同步单线程数据库，高并发场景下可能阻塞。
- **影响**：当前为单用户应用，影响极小。如果未来支持多用户同时操作，需要考虑。
- **建议**：如果需要支持多用户，考虑迁移到 PostgreSQL 或使用 libsql。

### 6. 缺乏用户认证
- **问题**：API 无任何认证机制，任何人访问到服务器即可操作数据。
- **影响**：部署到公网后数据不安全。
- **建议**：添加简单的 API Key 或 JWT 认证。

### 7. 缺乏数据分页
- **问题**：`GET /api/tasks` 返回所有任务，无分页。
- **影响**：任务数量超过数百条后响应变慢。
- **建议**：添加 `?page=&limit=` 分页参数。

## 改进方向

### 短期改进
1. **任务模板**：常用任务类型（如"混凝土浇筑"、"钢筋检测"）保存为模板，快速创建
2. **日程复制**：支持复制某天的日程到其他日期
3. **导出功能**：导出周历为 PDF 或图片
4. **搜索功能**：按关键词搜索任务和日程
5. **通知提醒**：日程开始前推送通知（浏览器 Notification API）

### 中期改进
1. **多项目管理**：当前为扁平任务列表，可改进为项目 → 任务层级
2. **团队协作**：支持多用户，分配任务给具体人员
3. **甘特图视图**：以甘特图形式展示任务时间线
4. **文件附件**：为任务/日程添加照片、文档附件
5. **数据导入/导出**：支持 Excel/CSV 导入导出

### 长期改进
1. **离线支持**：前端使用 Service Worker + IndexedDB 实现完整离线操作
2. **移动端原生应用**：使用 React Native 或 Tauri 打包为桌面/移动应用
3. **AI 辅助**：根据历史数据自动推荐日程安排
4. **微信小程序**：适配微信小程序，方便现场人员使用
5. **数据同步**：支持多设备间数据同步

## 前端技术债务

| 项目 | 描述 | 优先级 |
|------|------|--------|
| 组件拆分 | WeekCalendar.tsx 过大（>1000行），建议拆分为 DayColumn、EventList、Header 等子组件 | 中 |
| 状态管理 | 当前使用 React hooks，如功能增加可考虑 Zustand/Redux | 低 |
| 样式管理 | inline style 维护困难，可考虑 CSS Modules 或 Tailwind | 低 |
| 类型安全 | 部分 API 响应使用 `any` 类型，应定义完整接口 | 中 |
| 错误处理 | API 调用失败时缺少全局错误提示 | 中 |

## 后端技术债务

| 项目 | 描述 | 优先级 |
|------|------|--------|
| 输入验证 | 缺少请求体字段类型/长度验证 | 高 |
| 错误处理 | 错误响应格式不统一 | 中 |
| 日志记录 | 无结构化日志，排查问题困难 | 中 |
| API 文档 | 缺少 OpenAPI/Swagger 文档 | 低 |
| 测试覆盖 | 无单元测试和集成测试 | 高 |
